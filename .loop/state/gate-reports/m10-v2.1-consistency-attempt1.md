**fourth, load-bearing source the doc omits entirely**:
- `src/paperproof/docsdb/coverage.py:408-418` — source **(iv) S2 critic coverage

  critic's `angle_covered` verdict onto PRODUCTIVE/tried_blocked/…).
- `grep -ic critic docs/17 == 0`: docs/17 never mentions critic reports /
  `coverage_report` / `angle_covered`. Source (iv) is undocumented in the fold.
- This is not cosmetic: for a **waved** target, (iv) is the ONLY path to
  `productive`/`tried_blocked`. Code source (i) (`coverage.py:358-369`) only ever
  yields `TRIED_EMPTY` and reads `mem["angle"]`, NOT the "plans + query_logs" the
  doc describes. A reader implementing saturation from docs/17 as written would
  compute a **different ledger** than the code for every waved node.
- The migrated saturation tests (`test_v_cov.py::test_saturation_floor_*`) rely on
  `_seed_critic_report(...)` to earn angles — confirming (iv) is the real mechanism,
  not dead code.
- Remediation: add source (iv) to docs/17's fold list and correct (i) to match
  `coverage.py` (member `angle` → tried_empty), **or** change the code. Doc-side
  fix is the natural one; it must land in the same change.
